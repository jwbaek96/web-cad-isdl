## 1. 프로젝트 목표

이번 리뉴얼의 핵심 목표는 두 가지다.

1. 사이트가 예쁘고 잘 만들어져 보이게 할 것
2. 의뢰인이 일부 콘텐츠를 쉽게 관리할 수 있게 할 것

즉, 대규모 기능 개발이나 복잡한 시스템 구축보다, 시각적 완성도와 운영 편의성을 균형 있게 확보하는 것이 중요하다.

---

## 2. 리뉴얼 방향

### 2.1 디자인 방향

- 오래된 연구실 사이트 느낌을 줄이고, 정돈되고 현대적인 인상으로 개선
- 과장된 연출보다 차분하고 신뢰감 있는 톤 유지
- 모바일에서도 깔끔하게 보이는 반응형 화면 구성 적용
- 텍스트가 많은 사이트 특성을 고려해 가독성과 정렬감 중심으로 디자인

### 2.2 색상 방향

- 리뉴얼 사이트의 주요 색상은 중앙대학교 전용색상을 기반으로 활용
- 학교 아이덴티티를 반영하되, 사이트 전체가 너무 무겁거나 딱딱해 보이지 않도록 비중을 조절
- 강한 포인트 컬러는 제한적으로 사용하고, 전체 인상은 블루/그레이 계열 중심으로 안정감 있게 구성

권장 활용안:

1. 메인 컬러
- CAU BLUE: RGB 33, 85, 164 / Hex #2155A4
- 사이트의 대표 색상으로 사용
- 헤더 포인트, 주요 버튼, 링크 강조, 섹션 타이틀 등에 적합

2. 서브 컬러
- CAU GRAY: RGB 102, 102, 102 / Hex #666666
- 본문 보조 텍스트, 구분선, 정보성 블록에 적합

- CAU LIGHT GRAY: RGB 230, 230, 230 / Hex #E6E6E6
- 배경면, 카드 구분, 표/리스트의 약한 경계 처리에 적합

3. 포인트 컬러
- CAU RED: RGB 255, 0, 0 / Hex #FF0000
- 강조 배지, 중요 공지, 아주 제한적인 CTA 포인트로만 사용
- 과도하게 쓰면 연구실 사이트보다 홍보 페이지처럼 보일 수 있으므로 사용량을 낮게 유지

4. 보조 포인트
- CAU SILVER, CAU GOLD는 필요 시 성과/수상/특별 섹션에만 제한적으로 활용
- 기본 화면 전반의 주조색으로 쓰기보다는 장식적 포인트에 가깝게 적용

색상 운용 원칙:

- 기본 배경은 화이트 또는 아주 옅은 그레이 톤 유지
- 전체 인상은 CAU BLUE + 화이트 + 라이트 그레이 조합으로 정리
- CAU RED는 핵심 강조 1~2군데에만 사용
- 연구실 사이트인 만큼 화려함보다 정제된 완성도를 우선

### 2.3 구조 방향

- 기존 사이트의 큰 메뉴 구조는 유지
- 다만 메뉴명, 화면 우선순위, 정보 배치를 정리해 더 이해하기 쉽게 개선
- 페이지 수를 무리하게 늘리지 않고, 핵심 정보가 잘 보이도록 구성

### 2.4 운영 방향

- 사이트 배포는 GitHub 기반으로 진행
- 전체 CMS 대신 필요한 항목만 관리 가능하게 구성
- Google Sheets 또는 Google Apps Script는 자주 바뀌는 데이터에 한해 제한적으로 사용

## 3. 우선순위

### 3.1 가장 중요한 것

1. 첫인상이 좋아 보일 것
2. 연구실의 성과와 신뢰감이 잘 드러날 것
3. 모바일에서도 불편하지 않을 것
4. 의뢰인이 자주 수정하는 정보는 쉽게 바꿀 수 있을 것

### 3.2 이번 단계에서 과하지 않게 가져갈 것

- 복잡한 관리자 시스템
- 과도한 인터랙션
- 대규모 검색/필터 기능
- 모든 페이지의 동적 관리 기능

## 5. 핵심 페이지 기획 메모

### 5.1 Home

- 연구실 정체성을 바로 보여주는 첫 화면 필요
- 대표 소개 문구, 연구 키워드, 최신 소식, 주요 성과를 요약해서 배치
- 방문자가 빠르게 연구실 성격을 파악할 수 있어야 함

### 5.2 Research

- 연구 주제를 시각적으로 정리
- 텍스트 덩어리보다 카드형 또는 섹션형 구성이 적합
- 현재 진행 중인 프로젝트와 대표 연구 분야 중심으로 구성

### 5.3 Publications

- 연구실의 신뢰도를 보여주는 핵심 페이지
- 긴 텍스트 나열보다 연도 기준 정리 또는 보기 쉬운 리스트 구성이 필요
- 필요 시 이 영역은 시트 연동 우선 후보로 고려 가능

### 5.4 Members

- 교수, 재학생, 졸업생 정보를 정돈된 형식으로 배치
- 졸업생 진출 현황은 연구실 신뢰도에 중요한 요소이므로 잘 보이게 구성
- 멤버 정보 역시 시트 관리 후보가 될 수 있음

### 5.5 News / Activities

- 자주 바뀌는 항목이므로 관리 편의성이 중요
- 시트 연동이 가장 유효한 페이지 후보
- 너무 긴 아카이브보다 최신성 전달이 중요

### 5.6 Education

- 자료실 성격이 강하므로 보기 좋게 목록화하는 것이 중요
- PDF/HWP/XLS 링크를 정돈된 형태로 정리하면 인상이 크게 좋아질 수 있음

## 6. 관리 편의성 구상

### 6.1 정적으로 관리할 항목

- 레이아웃
- 디자인 스타일
- 고정 소개 문구
- 메뉴 구조
- 연구실 기본 정보

### 6.2 쉽게 수정 가능하게 만들 항목

- 뉴스
- 논문 목록
- 구성원 목록
- 공지성 링크

### 6.3 권장 방식

- 기본 구조는 정적 사이트로 제작
- 자주 바뀌는 데이터만 Google Sheets와 연결
- Google Apps Script는 필요할 때만 최소한으로 적용

이 방식은 디자인 품질을 유지하면서도 운영 부담을 낮추는 데 적합하다.

## 10. 한 줄 정리

이번 리뉴얼은 구조를 크게 뒤엎는 프로젝트가 아니라, 기존 콘텐츠를 바탕으로 사이트를 더 예쁘고 세련되게 보이게 만들고, 일부 콘텐츠는 더 쉽게 관리하게 만드는 프로젝트로 정의하는 것이 적절하다.
