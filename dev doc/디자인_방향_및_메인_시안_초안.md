# ISDL 홈페이지 디자인 방향 및 메인 시안 초안

- 작성일: 2026-07-10
- 목적: 리뉴얼 제작에 들어가기 전, 최소한의 디자인 방향을 정하고 메인 화면의 초안 구조를 빠르게 확정하기 위한 문서
- 전제: 이번 프로젝트는 대규모 기획보다 "좋아 보이는 완성도"와 "현실적인 제작/운영"이 중요함

---

## 1. 이번 단계에서 먼저 정할 것

사이트 제작 전에 길게 고민할 필요는 없지만, 아래 5가지는 먼저 정하고 들어가는 것이 좋다.

1. 전체 분위기
- 차분한지
- 조금 더 현대적인지
- 학술기관답게 단정한지

2. 색상 사용 비율
- CAU BLUE를 어느 정도 강하게 쓸지
- CAU RED를 얼마나 절제할지

3. 첫 화면 메시지
- 연구실 소개 중심
- 연구 분야 중심
- 성과 요약 중심

4. 정보 밀도
- 여백이 넉넉한지
- 정보가 촘촘한지

5. 메인 화면 우선순위
- 소개
- 연구 분야
- 주요 논문
- 멤버
- 뉴스

이 다섯 가지만 정해도 바로 시안 제작과 퍼블리싱으로 넘어갈 수 있다.

---

## 2. 권장 디자인 방향

현재 프로젝트에 가장 적합한 방향은 다음과 같다.

### 권장안: 단정하고 현대적인 연구실 사이트

핵심 키워드:
- 차분함
- 신뢰감
- 정돈감
- 학술적 이미지
- 오래 봐도 질리지 않는 디자인

방향 요약:
- 메인 컬러는 CAU BLUE를 사용
- 배경은 화이트와 라이트 그레이 위주로 구성
- CAU RED는 극소수 포인트에만 사용
- 큰 사진보다 타이포와 정보 정렬로 완성도를 만드는 방식
- 카드형 레이아웃과 충분한 여백으로 현대적인 인상 형성

이 방향이 좋은 이유:
- 연구실 사이트에 맞는 신뢰감을 유지할 수 있음
- 제작 난이도가 지나치게 올라가지 않음
- GitHub 기반 정적 사이트와도 잘 맞음
- 나중에 페이지를 늘려도 일관성 유지가 쉬움

## 3. 메인 시안 방향 3가지

### 시안 A. Academic Clean

가장 무난하고 안정적인 방향이다.

특징:
- 흰 배경 중심
- CAU BLUE 포인트 사용
- 텍스트 중심의 정돈된 레이아웃
- 카드형 섹션과 얇은 경계선 활용

첫인상:
- 단정하다
- 깔끔하다
- 믿음직하다

적합한 경우:
- 교수님이나 연구실 측에서 지나치게 트렌디한 느낌보다 안정감을 원할 때
- 오래 써도 무난한 디자인이 필요할 때

### 시안 B. Research Showcase

성과와 연구 분야를 더 강조하는 방향이다.

특징:
- 첫 화면에 연구 키워드와 성과 숫자를 배치
- 프로젝트/논문/진출 현황을 시각적으로 요약
- 이미지와 숫자 블록 비중이 조금 더 큼

첫인상:
- 활동적이다
- 연구실 역량이 강해 보인다
- 대외 홍보에 유리하다

적합한 경우:
- 산학협력, 지원자 유입, 연구실 브랜딩을 더 적극적으로 보여주고 싶을 때

### 시안 C. Modern Editorial

타이포와 여백을 더 강조한 방향이다.

특징:
- 큰 제목과 간결한 문장 중심
- 정보량은 많지만 화면은 여유롭게 보이게 구성
- 뉴스/논문/연구 분야를 에디토리얼 스타일로 배치

첫인상:
- 세련됐다
- 요즘 느낌이 난다
- 완성도 있어 보인다

적합한 경우:
- 기존 대학 사이트보다 조금 더 현대적인 인상을 원할 때
- 첫인상에서 확실히 바뀐 느낌을 주고 싶을 때

### 추천

현재 프로젝트에는 시안 A와 시안 B를 섞는 방식이 가장 적합하다.

추천 이유:
- 기본 톤은 Academic Clean처럼 안정적으로 가져가고
- 첫 화면과 성과 영역만 Research Showcase처럼 조금 더 힘을 주면
- 과하지 않으면서도 분명히 좋아 보이는 결과를 만들 수 있다.

## 4. 메인 페이지 구조 초안

### 4.1 첫 화면

첫 화면에는 아래 요소가 들어가는 것이 좋다.

- 연구실명
- 한 줄 소개
- 대표 연구 키워드 3~5개
- 간단한 설명 문단
- 주요 버튼 2개
- 대표 이미지 또는 연구실 관련 시각 요소

권장 버튼:
- Research 보기
- Publications 보기

선택 가능 버튼:
- Join Us
- Contact

첫 화면 예시 구성:
- 좌측: 연구실명, 소개 문구, 버튼
- 우측: 연구 이미지 또는 추상적 그래픽

### 4.2 주요 성과 요약 영역

첫 화면 아래에는 짧은 성과 요약 구간이 있으면 좋다.

예시 항목:
- Research Projects
- Publications
- Alumni
- Active Members

이 영역은 숫자를 정확히 확정할 수 있을 때 넣는 것이 좋고, 아직 데이터가 정리되지 않았다면 후순위로 둘 수 있다.

### 4.3 연구 분야 영역

권장 방식:
- 3~6개 정도의 연구 분야를 카드형으로 정리
- 각 카드에는 제목, 짧은 설명, 관련 키워드 배치

이유:
- 방문자가 연구실이 무엇을 하는지 가장 빠르게 이해할 수 있음

### 4.4 최신 논문 또는 주요 논문 영역

권장 방식:
- 최근 논문 3~5개만 먼저 노출
- 전체 목록은 Publications 페이지에서 확인

이유:
- 메인 페이지가 너무 길어지는 것을 막을 수 있음
- 연구실 신뢰도를 빠르게 전달할 수 있음

### 4.5 멤버 또는 졸업생 진출 현황 영역

권장 방식:
- 교수 소개 + 재학생/졸업생 일부 하이라이트
- 졸업생 진출처를 짧게 보여주는 블록 구성 가능

이유:
- 지원자 입장에서 매우 중요한 정보임
- 연구실 신뢰감 형성에 효과적임

### 4.6 뉴스 / 활동 영역

권장 방식:
- 최근 3~4개만 노출
- 너무 긴 아카이브는 서브 페이지로 분리

이유:
- 사이트가 살아 있다는 인상을 줌
- 운영 편의성 측면에서도 시트 연동에 적합함

### 4.7 푸터

권장 정보:
- 연구실명
- 소속
- 연락처
- 이메일
- 주소
- 관련 링크

## 5. 모바일 기준 원칙

모바일에서는 아래 원칙을 지키는 것이 중요하다.

1. 첫 화면 문장이 너무 길지 않을 것
2. 카드가 너무 작아지지 않을 것
3. 버튼은 손으로 누르기 쉬운 크기일 것
4. 텍스트와 링크가 빽빽하지 않을 것
5. 논문/뉴스 목록은 요약 중심으로 보여줄 것

즉, 데스크톱에서 예쁜 것보다 모바일에서 보기 좋은 것이 더 중요하다.

## 6. 바로 제작에 들어갈 때의 추천 순서

1. 메인 페이지부터 먼저 디자인 및 퍼블리싱
2. Research 페이지 제작
3. Publications 페이지 제작
4. Members 페이지 제작
5. News / Education / Contact 순서로 확장

이 순서가 좋은 이유:
- 메인 페이지에서 전체 톤이 먼저 정해짐
- 핵심 페이지 3~4개만 만들어도 사이트의 완성도가 빠르게 올라감
- 나머지 페이지는 그 스타일을 확장하면 됨

## 7. 현재 기준 추천 결정안

지금 바로 결정한다면 아래처럼 가는 것이 가장 무난하다.

- 디자인 톤: 단정하고 현대적
- 메인 컬러: CAU BLUE 중심
- 포인트 컬러: CAU RED 최소 사용
- 메인 화면: 연구실 소개 + 연구 분야 + 최신 논문 + 뉴스
- 스타일: 카드형 레이아웃 + 넉넉한 여백 + 깔끔한 타이포
- 운영 방식: 정적 사이트 + 일부 시트 연동

이 프로젝트는 디자인 방향을 짧게 정한 뒤, 메인 페이지부터 바로 제작하면서 다듬는 방식이 가장 효율적이다.
