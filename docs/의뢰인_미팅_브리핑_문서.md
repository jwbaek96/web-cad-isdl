# ISDL 홈페이지 리뉴얼 의뢰인 미팅 브리핑 문서

- 작성일: 2026-07-10
- 목적: 의뢰인 미팅에서 설명해야 할 내용과, 반드시 확인/수집해야 할 정보를 정리한 실무용 문서
- 전제: 이번 리뉴얼은 대규모 시스템 구축보다 시각적 완성도 향상과 최소한의 운영 편의성 확보가 목표

---

## 1. 이번 미팅의 목표

이번 미팅에서는 아래 3가지를 명확히 하는 것이 중요하다.

1. 의뢰인이 원하는 리뉴얼의 수준과 방향 확인
2. 실제로 누가 어떤 콘텐츠를 관리할지 확인
3. 제작 범위와 우선순위를 정할 수 있는 기초 정보 확보

즉, 이번 미팅은 디자인 취향만 듣는 자리가 아니라, 사이트의 실제 운영 방식과 의사결정 기준을 받아내는 자리여야 한다.

---

## 2. 의뢰인에게 브리핑해야 할 내용

### 2.1 현재 사이트 진단 요약

의뢰인에게는 아래 정도로 간단히 설명하면 된다.

- 현재 사이트는 콘텐츠 양과 연구 성과는 충분함
- 문제는 내용 부족이 아니라, 사이트가 오래되고 정돈되지 않아 보인다는 점임
- 따라서 이번 리뉴얼은 구조를 크게 뒤엎기보다, 더 세련되고 신뢰감 있게 보이도록 개선하는 데 초점을 둠
- 동시에 일부 콘텐츠는 의뢰인이 쉽게 관리할 수 있는 구조를 고려 중임

### 2.2 리뉴얼 목표

- 사이트가 예쁘고 잘 만들어져 보일 것
- 중앙대학교 및 연구실의 신뢰감을 잘 보여줄 것
- 모바일에서도 보기 좋을 것
- 뉴스, 논문, 멤버 등 일부 정보는 관리가 덜 번거로울 것

### 2.3 디자인 방향

- 과장된 연출보다 차분하고 정돈된 방향
- 텍스트가 많은 연구실 사이트 특성상 가독성과 정보 정리가 중요함
- 중앙대학교 전용색상을 적절히 반영하되, 전체적으로 너무 무겁거나 학교 본부 사이트처럼 딱딱해지지 않게 조절할 예정
- 전체 인상은 CAU BLUE, 화이트, 라이트 그레이 중심으로 안정감 있게 구성할 예정

### 2.4 구조 방향

- 기존 메뉴 체계를 크게 흔들지 않는 방향
- 다만 홈, 연구, 논문, 멤버, 뉴스, 교육 페이지의 우선순위와 표현 방식을 개선
- 핵심 정보가 빨리 보이고, 첫인상이 좋아지도록 구성할 예정

### 2.5 운영 방향

- 사이트는 GitHub 기반으로 배포 예정
- 전체 CMS를 넣기보다, 필요한 부분만 쉽게 관리 가능하게 할 예정
- Google Sheets 또는 Google Apps Script는 정말 필요한 영역만 제한적으로 활용할 예정

### 2.6 이번 단계에서 하지 않으려는 것

- 복잡한 관리자 시스템 구축
- 과도한 인터랙션이나 무거운 기능 개발
- 모든 콘텐츠를 동적으로 관리하는 구조
- 디자인보다 시스템이 더 커지는 방향

이 부분은 미리 설명해 두는 것이 좋다. 그래야 의뢰인이 초기에 과한 기대를 하지 않고, 현실적인 범위 안에서 결정을 내릴 수 있다.

## 3. 의뢰인에게 반드시 받아내야 할 정보

### 3.1 목표와 기대 수준

1. 이번 리뉴얼에서 가장 중요하게 생각하는 것은 무엇인가?
- 예쁨
- 신뢰감
- 지원자 유입
- 산학협력 홍보
- 운영 편의성

2. 현재 사이트에서 가장 불만인 점은 무엇인가?

3. 반대로 현재 사이트에서 반드시 유지하고 싶은 점은 무엇인가?

4. 이번 리뉴얼이 끝났을 때, 의뢰인이 가장 만족할 결과는 무엇인가?

### 3.2 디자인 관련 정보

1. 원하는 느낌이 더 클래식한지, 더 모던한지

2. 참고하고 싶은 다른 연구실/학교/기업 사이트가 있는지

3. 중앙대학교 전용색상 사용 비중에 대한 선호
- 학교 정체성이 강하게 드러나길 원하는지
- 연구실 독립성이 더 느껴지길 원하는지

4. 메인 화면에서 가장 먼저 보여주고 싶은 정보가 무엇인지
- 연구실 소개
- 대표 연구 분야
- 최신 뉴스
- 주요 논문
- 모집 정보

### 3.3 콘텐츠 관련 정보

1. 이번 리뉴얼에서 반드시 들어가야 할 페이지는 무엇인지

2. 반대로 정리하거나 삭제해도 되는 콘텐츠는 무엇인지

3. 최신 자료 기준으로 아래 항목의 정리본을 줄 수 있는지
- 연구실 소개 문구
- 교수 소개
- 연구 분야 설명
- 프로젝트 목록
- 논문 목록
- 구성원 목록
- 졸업생 현황
- 뉴스/활동 내역
- 교육자료 목록

4. 영문 페이지가 필요한지, 필요하다면 어느 범위까지 필요한지

### 3.4 운영 관련 정보

1. 오픈 후 실제로 사이트를 수정할 사람은 누구인지

2. 그 사람이 코드 수정이 가능한지, 아니면 문서/시트 수준 수정만 가능한지

3. 자주 바뀌는 콘텐츠가 무엇인지
- 뉴스
- 논문
- 멤버
- 공지
- 모집 정보

4. Google Sheets로 관리하는 방식에 거부감이 없는지

5. Google Apps Script 사용이 가능한 환경인지
- 구글 계정 사용 가능 여부
- 조직 내 정책상 외부 도구 사용 제한 여부

6. 배포 후 유지보수는 누가 맡을 예정인지

### 3.5 기술/운영 접근 정보

1. 현재 도메인과 호스팅은 누가 관리하는지

2. GitHub로 배포하는 데 문제가 없는지

3. 기존 사이트 파일이나 자료 원본을 전달받을 수 있는지

4. 기존 이미지, 로고, 문서 파일의 원본 보유 여부

5. 향후 도메인 연결 또는 이전이 필요한지

## 4. 있으면 바로 받아오는 것이 좋은 자료

미팅 자리에서 바로 요청하거나, 미팅 직후 전달받을 수 있게 정리해야 한다.

1. 연구실 로고 파일
2. 교수님 프로필 사진 및 구성원 사진
3. 대표 연구 이미지 또는 사용할 수 있는 사진 자료
4. 최신 논문 목록 파일
5. 최신 멤버 목록 파일
6. 졸업생 진출 현황 자료
7. 연구 과제/프로젝트 정리 자료
8. 교육자료 원본 목록
9. 기존 사이트 수정 권한 또는 원본 파일 접근 가능 여부
10. 중앙대학교 관련 브랜딩 자료 사용 가능 범위

---

## 5. 미팅 중 꼭 확인해야 할 의사결정 포인트

### 5.1 범위 확정용 질문

- 이번에 우선 바꿀 페이지는 어디까지인지
- 전체 페이지를 모두 손볼지, 핵심 페이지 위주로 먼저 정리할지
- 영문 페이지를 동시에 진행할지

### 5.2 운영 방식 확정용 질문

- 누가 직접 수정할 것인지
- 무엇을 직접 수정하고 싶은지
- 시트 기반 관리가 실제로 편한지

### 5.3 디자인 방향 확정용 질문

- 학교 공식 느낌에 가까운 쪽이 좋은지
- 연구실만의 분위기가 조금 더 드러나는 쪽이 좋은지
- 차분한 쪽이 좋은지, 조금 더 현대적인 느낌이 좋은지

---

## 6. 미팅 진행 순서 추천

1. 현재 사이트의 문제를 짧게 요약
2. 이번 리뉴얼 목표를 간단히 설명
3. 디자인 방향과 운영 방향을 설명
4. 의뢰인의 우선순위와 기대 수준을 질문
5. 운영 주체와 관리 방식 확인
6. 필요한 자료와 접근 권한 요청
7. 다음 단계 일정과 산출물 합의

---

## 7. 미팅 후 정리해야 할 항목

미팅이 끝나면 아래 내용을 바로 정리해야 한다.

1. 의뢰인이 가장 중요하게 본 우선순위
2. 확정된 페이지 범위
3. 직접 관리가 필요한 항목
4. 제공받기로 한 자료 목록
5. 추가 확인이 필요한 미해결 이슈
6. 다음 미팅 또는 시안 공유 일정

---

## 8. 한 줄 정리

이번 미팅의 핵심은 "예쁘게 만드는 방향을 설명하는 것"과 "누가 무엇을 어떻게 관리할지에 대한 현실적인 정보를 받아오는 것"이다.
